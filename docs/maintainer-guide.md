# Maintainer Guide

Maintainers triage tester reports before filing anything upstream into `openclaw/openclaw`.

## Triage rules

- Search duplicates first.
- Confirm the report is about the target tag or a relevant baseline.
- Ask for missing evidence before filing upstream when reproduction is weak.
- File one upstream issue per confirmed bug.
- Use `Beta blocker: <component> - <summary>` for release-blocking issues.
- Backlink upstream issues to the public tester report and run folder.
